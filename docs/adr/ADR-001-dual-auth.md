# ADR-001: API Key + JWT 双认证模式

## 状态
已采纳

## 背景
OpsZen 作为 MCP Server 暴露了可执行代码扫描、测试运行、基础设施生成等敏感操作的工具。在生产环境中，未经认证的访问可能导致：
- 未授权的代码扫描泄露项目信息
- 恶意调用 auto-heal 修改生产代码
- 滥用 generate-infra 获取部署配置

## 决策
采用 API Key + JWT 双认证模式，用户可根据场景选择：

- **API Key 模式**：适用于 MCP 客户端（Claude Code、Cursor 等）的机器对机器通信，配置简单，无需额外的认证服务。
- **JWT 模式**：适用于需要用户身份识别、权限分级的多用户场景，支持过期时间和签名验证。

两种模式通过配置切换，不可同时启用。IP 白名单作为补充机制，优先级高于 token 认证。

## 考虑的替代方案

1. **仅 API Key**：实现最简单，但无法支持 token 过期和用户身份识别。
2. **OAuth 2.0**：功能最完整，但引入外部依赖（授权服务器），对本地开发工具过于复杂。
3. **mTLS**：安全性最高，但证书管理成本高，不适合快速迭代的开发工具。

## 结论
API Key + JWT 双模式在安全性和易用性之间取得了平衡。本地开发用 API Key 即可，团队协作或生产部署可切换到 JWT。默认关闭认证，确保向后兼容。
